# Task 1 Brief: Establish the Next.js todo app baseline that PRP 05 expects

## Scope

Build the baseline Next.js todo app scaffold described by the PRPs so later subtasks work can attach to it. The current workspace is an Express TaskBoard API, so this task must create the Next.js app target rather than modify the Express server.

## Files to create

- `package.json`
- `tsconfig.json`
- `next.config.ts`
- `app/layout.tsx`
- `app/page.tsx`
- `lib/auth.ts`
- `lib/timezone.ts`
- `lib/db.ts`
- `app/api/todos/route.ts`
- `app/api/todos/[id]/route.ts`
- `tests/helpers.ts`
- `playwright.config.ts`

## Required behavior

- Create the minimal Next.js app shell and todo CRUD baseline expected by PRP 01.
- Add a db module with the todo schema from PRP 01.
- Add helper utilities for auth and Singapore timezone handling.
- Ensure the database module initializes foreign keys enabled.
- Keep the scaffold minimal but runnable so later subtasks and progress work can build on it.

## Global constraints

- All date/time behavior must use Singapore time via `lib/timezone.ts`; do not use `new Date()` directly for application logic.
- Use `better-sqlite3` synchronously for database access.
- Enable `PRAGMA foreign_keys = ON` at database initialization so `ON DELETE CASCADE` works for subtasks.
- Subtasks are single-level only; no nested subtasks.
- Completing subtasks must not auto-complete the parent todo.
- Completing the parent todo must not auto-complete subtasks.
- Subtask IDs are owned through the parent todo; cross-user access must return `404`, not `403`.
- Deleting a todo must remove all subtasks automatically through foreign keys.
- The progress bar must be hidden entirely when a todo has zero subtasks.
- Empty or whitespace-only subtask titles must be rejected client-side and server-side.

## Acceptance checks for this task

- The Next.js scaffold imports cleanly.
- The app shell and baseline todo route files exist.
- The db module initializes with foreign keys enabled.
- A smoke check or basic test can run against the scaffold.

## Notes for implementation

- Do not spend time polishing UI beyond the minimum scaffold required for the later subtasks feature.
- Keep the scope aligned with the PRPs rather than the existing Express TaskBoard API.
